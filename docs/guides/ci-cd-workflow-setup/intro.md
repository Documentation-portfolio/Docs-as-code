## Introduction

A CI/CD workflow ensures that every documentation update is validated, tested, and deployed in an automated manner.  
In a Docs-as-Code approach, documentation is treated the same way as source code, that meaans formatting checks, linting, link validation, and automated deployment become essential parts of the publishing pipeline.

The following procedures help you set up the foundational workflows required for a reliable documentation pipeline.

- [Validation Workflow — Quality Gate](/docs/guides/validation)
- [Deployment Workflow — Publish to GitHub Pages](/docs/guides/deployment)
- [PR Preview Workflow — Live PR Previews](/docs/guides/pr-preview)
