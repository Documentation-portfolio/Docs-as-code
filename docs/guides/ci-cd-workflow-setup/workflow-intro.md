## Introduction

A Continuous Integration and Continuous Deployment (CI/CD) workflow ensures that every documentation update is validated, tested, and deployed in an automated manner.  
In Docs-as-Code approach, documentation is treated the same way as source code, that meaans formatting checks, linting, link validation, and automated deployment become essential parts of the publishing pipeline.

The following procedures help you set up the foundational, required workflows:

- [Validation Workflow](/docs/guides/validation)
- [Deployment Workflow](/docs/guides/deployment)
- [PR Preview Workflow](/docs/guides/pr-preview)
